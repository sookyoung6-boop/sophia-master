핵심 원칙
원칙 01
소비자는 제품이 아니라 변화를 산다
Brian Tracy의 원칙. 모든 기획과 전략의 출발점. 제품이 아니라 그 제품이 만드는 삶의 변화를 먼저 설계한다.
원칙 02
COO는 일을 하는 사람이 아니라 흐름을 관리하는 사람
직접 처리율이 높아지는 순간이 구조 붕괴 신호다. 내가 손대기 전에 위임 구조를 먼저 점검한다.
원칙 03
즉흥은 리스크다
급한 느낌 = 위험 신호. 화난 상태의 메시지, 수면 부족의 결정, 확신 100%의 판단은 모두 멈추는 기준이다.
원칙 04
기록이 자산이다
결정한 것, 위임한 것, 배운 것을 모두 남긴다. 언젠가 이 기록이 새로운 팀원의 온보딩 문서가 된다.
Chapter 01
프로젝트를
시작할 때
모든 프로젝트는 명확한 정의에서 시작됩니다. 시작 단계에서 흐릿한 것은 끝까지 흐립니다.
01 프로젝트 정의 5문항
이 5가지 질문에 답할 수 없으면 프로젝트를 시작하지 않습니다.
질문작성 예시
이 프로젝트의 목적은 무엇인가?쓰리포 IP를 활용한 힐링 굿즈 브랜드 런칭. 20-30대 직장인의 일상 회복 경험 제공.
끝나면 무엇이 남는가? (자산화)IP 굿즈 라인업 3종, 브랜드 포트폴리오, 온라인 판매 채널 구축.
결정권자는 누구인가?최종: 대표님. 실무: 소피아. 디자인: 디자인팀장.
성공 기준은 무엇인가?런칭 3개월 내 손익분기점 달성 / 재구매율 20% 이상.
실패로 보는 기준은 무엇인가?런칭 후 재고 70% 이상 / 브랜드 세계관 훼손 / 팀 번아웃 발생.
02 프로젝트 킥오프 체크리스트
  • ✓ 프로젝트명 확정 (내부용 코드명 포함)
  • ✓ 목적 / 자산화 목표 문서화
  • ✓ 담당자 배정 (디자인 / 콘텐츠 / 운영 각 1인)
  • ✓ 전체 타임라인 초안 작성 (마일스톤 포함)
  • ✓ 예산 범위 확인 및 대표 승인
  • ✓ 외부 파트너/외주 여부 결정
  • ✓ 킥오프 미팅 일정 확정 및 아젠다 배포
  • ✓ 프로젝트 드라이브 폴더 생성 및 구조 설정
  • ✓ 1차 컨펌 시점 대표님과 사전 합의

Chapter 02
기획
단계
기획은 방향을 잡는 단계입니다. 이 단계에서 흔들리면 이후 모든 단계가 흔들립니다.
01 기획 브리프 필수 항목
항목설명작성 책임
브랜드 세계관이 프로젝트가 브랜드 전체 세계관에서 어디에 위치하는가COO / 대표
타깃 고객구체적인 페르소나 (연령, 라이프스타일, 구매 동기)COO
핵심 감정이 제품/콘텐츠를 받았을 때 고객이 느껴야 할 감정 1가지COO / 디자인팀
레퍼런스방향성 레퍼런스 3-5개 (이유 포함)디자인팀
금지 방향절대 하지 말아야 할 것 (톤, 색상, 표현 방식)COO
산출물 정의최종 결과물의 형태, 수량, 포맷COO / 운영팀
컨펌 기준무엇이 완성으로 보이는가 (수치 or 기준)COO
02 기획 단계 체크리스트
  • ✓ 브랜드 브리프 작성 완료 및 팀 공유
  • ✓ 타깃 페르소나 정의 (최소 1인)
  • ✓ 경쟁사/레퍼런스 분석 (3개 이상)
  • ✓ 핵심 감정 키워드 합의 (팀 내)
  • ✓ 금지 방향 문서화
  • ✓ 기획안 대표님 1차 보고 및 피드백 반영
  • ✓ 수정 범위 및 횟수 사전 합의
  • ✓ 외주/파트너가 있다면 브리프 전달 완료
03 기획 검토 필터 (진행 전 필수)
아래 4가지를 통과하지 못한 기획은 다음 단계로 넘기지 않습니다.
  • ✓ 지속성 — 3년 뒤에도 브랜드 방향과 일치하는가?
  • ✓ 정체성 — Spextrum의 세계관 (굿즈/캐릭터/공간) 중 하나에 속하는가?
  • ✓ 자산화 — 이 프로젝트가 끝나도 남는 무언가가 있는가?
  • ✓ 지금 시점 — 지금 해야 할 이유가 있는가? (트렌드, 예산, 팀 역량)
Chapter 03
디자인
검수 기준
디자인 검수는 취향이 아니라 기준으로 합니다. 감각적 판단 이전에 구조적 체크가 먼저입니다.
01 검수 전 확인 (COO → 디자인팀)
시안을 받기 전에 이 내용이 전달되었는지 확인합니다.
  • ✓ 브리프 전달 완료 여부 확인
  • ✓ 레퍼런스 공유 여부 확인
  • ✓ 금지 방향 전달 여부 확인
  • ✓ 산출물 포맷 / 사이즈 / 수량 합의 여부
  • ✓ 납기일 재확인
02 시안 1차 검수 기준
검수 항목기준판정
세계관 일치브랜드 무드보드 / 브리프와 방향이 맞는가필수
핵심 감정 전달브리프에서 정한 핵심 감정이 느껴지는가필수
금지 방향 준수절대 하지 말아야 할 것이 없는가필수
가독성텍스트가 있다면 읽히는가권장
실물 적용 가능성인쇄/제조 시 재현 가능한 색상과 형태인가필수
파일 정리 상태레이어 명칭, 폴더 구조가 정리되어 있는가권장
03 피드백 작성 원칙

피드백은 항상 이 구조로 작성합니다.

[잘된 것] — 구체적으로 무엇이 좋은지 먼저 말한다.
[수정 요청] — "느낌이 아니라" 이유와 방향을 함께 말한다.
[수정 범위] — 전체인지, 부분인지, 색상만인지 명확히 한다.
[다음 시안 기준] — 다음에 뭘 보면 OK인지를 명시한다.

❌ 하지 말아야 할 피드백: "좀 더 세련되게", "느낌이 달라", "뭔가 아쉬워" — 방향 없는 피드백은 팀을 지치게 합니다.
04 수정 횟수 원칙
단계기준
1차 시안방향성 검토. 전면 수정 가능. 단, 브리프 기준으로만 판단.
2차 시안디테일 수정. 전면 수정은 브리프가 바뀐 경우에만 허용.
3차 시안최종 확인. 오탈자, 컬러값, 파일 정리만 확인.
3차 이후브리프 또는 대표 지시 변경이 있는 경우에만 추가 수정 진행. 반드시 COO 판단.
05 최종 디자인 파일 납품 기준
  • ✓ 원본 파일 (AI / PSD / Figma) 정리 완료
  • ✓ 인쇄용 PDF / 웹용 PNG 각각 추출
  • ✓ 컬러 코드 문서화 (CMYK / RGB / HEX)
  • ✓ 폰트 아웃라인 처리 또는 폰트 파일 첨부
  • ✓ 드라이브 02_디자인 → 최종본 폴더에 업로드
  • ✓ 파일명 규칙 준수: [프로젝트]_[용도]_[날짜]_final

Chapter 06
커뮤니케이션
원칙
말하는 방식이 신뢰를 만듭니다. COO의 언어는 팀의 언어가 됩니다.
01 내부 커뮤니케이션 원칙
원칙 1
배경 → 결정 → 다음 액션
어떤 공유도 이 3단계 구조로 합니다. 결론만 말하거나 배경만 설명하는 커뮤니케이션은 팀을 혼란스럽게 합니다.
원칙 2
즉답 금지
대표/외부에서 즉답을 요청해도 "확인 후 드리겠습니다"가 먼저입니다. 즉답은 실수를 만듭니다.
원칙 3
감정 메시지 10분 룰
화가 난 상태에서 작성한 메시지는 발송 전 10분 대기. 재검토 후 발송. 이 룰을 어긴 메시지는 대부분 후회합니다.
원칙 4
약속은 서면으로

02 대표님 커뮤니케이션 원칙
대표님의 말은 '지시'가 아니라 '의도'입니다. COO는 그 의도를 팀이 실행 가능한 언어로 번역합니다.
상황원칙
보고할 때현황 → 판단 → 요청 사항 순서로. 결론을 먼저 말한다.
지시를 받았을 때즉시 수행보다 "제가 이렇게 이해했는데 맞나요?" 확인 먼저.
이견이 있을 때감정이 아닌 데이터와 이유로. "제가 다르게 보이는 부분이 있어서요."
문제가 생겼을 때숨기지 말고 빠르게. 문제 + 원인 + 대안을 함께 가져간다.
컨펌이 필요할 때선택지 2-3개를 제시하고 권장안을 명시. "A를 추천하는 이유는 ~입니다."
03 팀원 커뮤니케이션 원칙
상황원칙
업무 지시목적 + 범위 + 완료 기준 + 납기를 반드시 포함. "알아서 잘 해줘"는 금지.
피드백잘한 것 먼저 → 수정 요청 → 다음 기준 제시. SBI 구조 활용.
실수 발생 시책임 추궁보다 원인 파악. "왜 이렇게 됐는지" 먼저 이해.
병목 발생 시담당자 탓이 아닌 구조 문제로 접근. "어디서 막혔어요?"
칭찬/인정구체적으로. "잘했어요"보다 "이 부분이 특히 좋았어요."
04 외부 커뮤니케이션 원칙
상황원칙
제안/미팅 요청목적 + 기대 아웃풋 + 소요 시간을 미리 안내.
컨펌 요청마감일 명시 + 컨펌 방식 명시. 기다리지 않고 D-2에 리마인드.
거절이유 + 대안 제시. 단순 거절은 관계를 끊는다.
계약/협의내부 컨펌 없이 확정 발언 금지. 반드시 "내부 확인 후 연락드리겠습니다."

기준과 방식
위임은 손 놓는 것이 아닙니다. 구조를 만들고, 기준을 주고, 흐름을 보는 것입니다.
01 위임 결정 기준
이 중 하나라도 해당하면 위임합니다. 직접 처리율이 60% 넘으면 구조를 점검합니다.
  • ✓ 반복되는 업무인가? (같은 일을 3번 이상 직접 했다면 위임 구조를 만들 시점)
  • ✓ 담당자가 더 잘할 수 있는 영역인가?
  • ✓ 내가 하지 않아도 기준이 전달 가능한가?
  • ✓ 지금 내 시간이 더 가치 있는 곳에 쓰여야 하는가?
02 위임카드 작성 기준

위임카드는 반드시 이 6가지를 포함합니다.

1. 업무명 — 무엇을 해야 하는가
2. 세부 업무 — 구체적으로 어떤 것들인가
3. 완료 기준 — 이것이 되면 완료로 본다
4. 실패 기준 — 이것이 없으면 실패다
5. 데드라인 — 언제까지
6. 컨펌 방식 — 어떻게 확인받는가

01 굿즈 브랜드
핵심 리스크
재고 과다 / 브랜드 세계관 훼손
제조 수량은 보수적으로. 브랜드 일관성을 위해 COO가 최종 디자인 컨펌.
핵심 산출물
브랜드 브리프 + 제조 샘플 + 상세페이지
이 3가지가 없으면 런칭하지 않는다.
단계핵심 액션주의점
기획세계관 정의 → 페르소나 → 제품 컨셉트렌드 추종 금지. 세계관 우선.
디자인무드보드 → 시안 → 샘플인쇄/제조 가능 여부 초기에 확인
제조샘플 컨펌 → 발주 → 납기 관리납기 여유 2주 이상 확보
런칭채널 등록 → SNS → CS재고 수량 보수적으로

단계핵심 액션주의점
